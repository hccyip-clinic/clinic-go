# Clinic UI Redesign Wayfinder

## Destination

Produce an implementation-ready UI redesign specification for the full authenticated clinic app: a modern, professional, clean, and user-friendly layout with defined hierarchy, navigation, responsive behavior, visual tokens, component patterns, and usability acceptance criteria.

## Notes

The app is a single-practitioner Hong Kong clinic system using Go templates, HTMX, SQLite, and Tailwind CSS. Consult `CONTEXT.md`, the existing dashboard/navigation ADR, and the `refactoring-ui`, `web-typography`, and `codebase-design` skills. Prioritize desktop/laptop workflows with tablet support, English UI labels, Traditional Chinese data/content support, and a calm clinical workspace visual direction.

## Decisions so far

- The destination is an implementation-ready UI redesign spec, not an immediate code implementation.
- Scope covers the full authenticated app and core workflow screens.
- The visual direction is a calm clinical workspace with restrained color, clear typography, compact information density, and minimal motion.
- Desktop/laptop use is primary, with tablet support and a narrow-screen fallback.
- The UI is English-first while supporting Traditional Chinese patient and receipt content.
- Success means one-click receipt start, typical receipt completion under two minutes, visible location and next action, recoverable validation errors, keyboard usability, and readable contrast.
- [Establish the app shell and navigation hierarchy](001-app-shell-and-navigation.md) — use a responsive left sidebar with Primary and Administration groups, keep unavailable Reports out of navigation, and use page context, active states, and nested breadcrumbs for orientation.
- [Define the calm clinical visual system](002-visual-design-system.md) — use deep teal, warm-neutral surfaces, system typography with CJK fallback, constrained spacing and shape scales, semantic states, consistent outline icons, and restrained motion.
- [Shape receipt and patient workflow screens](003-workflow-screen-patterns.md) — keep receipt creation single-page and guided, make patient selection searchable, show live totals, stage lifecycle actions, prioritize dashboard quick actions, and preserve context through clear validation feedback.
- [Define responsive, accessible, and bilingual content rules](004-responsive-accessibility-content-rules.md) — target desktop, tablet, and narrow layouts explicitly; apply WCAG 2.2 AA practices; preserve readable tables/forms; and support Traditional Chinese data without truncation.
- [Define the bilingual receipt printout format](005-bilingual-receipt-printout-format.md) — print A4 portrait receipts with English-first paired Traditional Chinese labels for every human-facing field, preserving configured values and a clear clinic-to-total section order.

## Out of scope

- Changing backend behavior, data models, authentication, or business rules.
- Adding new product capabilities such as appointments, insurance, or payment integrations.
- Implementing the redesign in templates or CSS during the wayfinding stage.
